STEP 3 — ¼ Kelly仓位计算
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⚠️ 关键方法论要求：

使用多情景Kelly公式：f* = E[r] / Var[r]，其中r是5年总回报
不要用二元Kelly公式 f=(pb−q)/b，它假设"赢或亏光"，不适用于多情景
不要用年化IRR做Kelly，必须用5年总回报，因为年化会压缩离散度导致方差失真

3a. 计算5年总回报
对每个情景：rᵢ = 终值/$[X] − 1
3b. 计算 E[r]
E[r] = Σ(pᵢ × rᵢ)
3c. 计算 Var[r]
E[r²] = Σ(pᵢ × rᵢ²)
Var[r] = E[r²] − (E[r])²
3d. Full Kelly → ¼ Kelly
Full Kelly f* = E[r] / Var[r]
¼ Kelly = f* × 0.25
3e. 置信度折扣
加权平均置信度 = Σ(pᵢ × confidenceᵢ)
Raw Position = ¼ Kelly × 置信度
3f. IRR质量因子
Q = min(1, max(0, (E[IRR] − 4%) / 15%))
Q函数将E[IRR]的绝对水平转化为0-1之间的资本配置意愿：
E[IRR]Q值含义< 4%0不投——绝对回报不够覆盖机会成本，无论Kelly多好4%0刚到门槛，几乎不配仓位8%0.20有兴趣，小仓试探10%0.33可以建仓12%0.47标准配置区间15%0.67优秀，给大部分额度18%0.87接近满额度20%+1.00极为优秀，Kelly说多少就多少（受Cap限制）
Adjusted Raw = Raw × Q
Q的作用：

Kelly解决"每单位风险赚多少"（效率），Q解决"这个绝对回报值不值得占用资本"（门槛）
一个IRR=4%但方差极小的标的，Kelly会给出很大的Raw，但Q≈0直接拦截——你不应该用组合资本去赚国债利率
一个IRR=20%但方差大的标的，Kelly会给出较小的Raw，但Q=1不打折——绝对回报足够优秀，让Kelly自己管理风险
同一个Adjusted Raw = 同样的仓位：IRR=20%/低Kelly/低置信 和 IRR=10%/高Kelly/高置信 可以得到相同的Adjusted Raw → 相同仓位。这是合理的，因为前者"回报高但不确定"，后者"回报中等但很确定"

3g. 连续仓位映射
将Adjusted Raw通过指数压缩函数映射为最终仓位：
Position = Cap × (1 − e^(−Adjusted Raw / S))
参数设定：

Cap = 10%（单只个股的组合集中度硬顶，基于8-12只持仓的集中组合）
S = 12%（斜率参数，控制仓位对Adjusted Raw的敏感度）

映射参考表：
Adj Raw最终仓位信号含义< 2%~1.5%观察仓——正期望但conviction不足、方差过大、或IRR不够5%~3.4%试探仓——thesis初步成立，等待更多验证10%~5.7%标准仓——合理conviction，适度安全边际15%~7.1%重仓——Kelly、置信度、IRR三者都支持20%~8.1%高conviction——强安全边际 + 低方差 + 高IRR30%+~9.2%+趋近满仓——极高conviction，渐近Cap
函数特征：

凹函数：Adj Raw从3%→10%时仓位增长最快（奖励从"不确定"到"有conviction"的跃升）
递减边际：Adj Raw从20%→30%时仓位增长明显放缓（防止过度集中）
渐近收敛：无论Adj Raw多大，仓位不超过Cap=10%
连续可微：没有阶梯跳变，每一个百分点的改善都映射为仓位变化

3h. 最终仓位确定
最终仓位 = Cap × (1 − e^(−Adjusted Raw / S)) = 10% × (1 − e^(−Adj Raw / 0.12))
四层管道逻辑：

¼ Kelly → 风险调整后的数学最优下注比例（期望/方差）
置信度折扣 → 你对自身概率估计的可靠性校准
IRR质量因子Q → 绝对回报门槛（值不值得占用资本）
指数压缩映射 → 将Adjusted Raw压缩到合理的个股仓位区间，凹函数天然防止过度集中

每层解决一个独立问题，信息不重复，串联确保无损传导。

四层各自解决什么问题：
━━━━━━━━━━━━━━━━━━━━━━━━
¼ Kelly → 数学最优（给定赔率和概率分布，每单位风险的回报效率）
置信度 → 认知校准（你对自己估计有多可靠）
IRR质量因子Q → 绝对回报门槛（值不值得占用资本）
指数压缩 → 连续映射 + 集中度纪律（凹函数天然防止过度集中）
四者解决不同问题，信息不重复，串联确保无损传导。
为什么需要Q因子——纯Kelly的盲点：
━━━━━━━━━━━━━━━━━━━━━━━━
Kelly衡量的是"效率"：每单位风险赚多少。但效率高不等于值得配置资本。
反例：一个IRR=5%、方差极小的标的，Kelly给出很大的f*——
因为"每单位风险的回报"很高。但你不应该用组合资本去赚国债利率。
Q因子解决的问题：绝对回报太低时，即使风险回报效率高，也不值得占用资本。
为什么Q用E[IRR]而不是E[r]：

E[IRR]是年化指标，跨标的可比（5年持有期 vs 3年持有期的标的可以比较）
E[r]是总回报，受持有期长度影响，不适合做横向门槛
Q只需要一个"值不值得"的门槛判断，E[IRR]更直观

为什么Q和Kelly不是信息重复：

Kelly用的是E[r]和Var[r]（总回报的均值和方差），决定"下多大注"
Q用的是E[IRR]（年化回报绝对值），决定"这个注值不值得下"
两者的输入虽然相关（E[IRR]是E[r]的单调变换），但回答的问题完全不同
类比：Kelly是"赔率够不够"，Q是"这场比赛值不值得参加"

Q函数的设计理由：
━━━━━━━━━━━━━━━━━━━━━━━━
Q = min(1, max(0, (E[IRR] − 5%) / 15%))
为什么下限是5%：

当前无风险利率（10年期美债）约4-5%
IRR < 5%意味着期望回报不如国债，没有理由承担股票风险
5%是"机会成本门槛"，不是"这个标的不好"——可能thesis很好但估值不够便宜

为什么上限是20%：

20%+ IRR是极稀有的机会（5年翻2.5倍），不需要额外打折
如果一个标的的E[IRR]≥20%，应该让Kelly和置信度来决定仓位大小，Q不应该再限制
20%对应Q=1（满额度），意味着"绝对回报的门槛完全通过，只受Kelly和置信度约束"

为什么线性而非凹/凸：

线性最透明——你知道IRR每提高1%，Q增加0.067
凹函数（如√）会让中等IRR标的获得过多Q额度
凸函数（如²）会让中等IRR标的被过度惩罚
线性是无先验偏好的中立选择，实际使用后可以根据经验调整

调整指南：

如果你在低利率环境（2%无风险），可以把下限降到3%
如果你在高利率环境（7%无风险），可以把下限提到7%
下限应该≈无风险利率，上限可以保持20%不变
公式变为：Q = min(1, max(0, (E[IRR] − Rf) / (20% − Rf)))

"相同待遇"原则的验证：
━━━━━━━━━━━━━━━━━━━━━━━━
核心原则：同样的Adjusted Raw = 同样的仓位。
标的A：IRR=20%, Kelly小(高方差), 低置信度
→ Raw = 小 × 低 = 8%, Q = 1.00, Adj Raw = 8.0%
标的B：IRR=10%, Kelly大(低方差), 高置信度
→ Raw = 大 × 高 = 24%, Q = 0.33, Adj Raw = 8.0%
两者Adj Raw相同 → 仓位相同 → ≈4.9%
解释：A "回报高但不确定"，B "回报中等但很确定"。
Kelly已经在Raw层面交换了"回报 vs 确定性"，
Q在此基础上加入"绝对回报值不值得"的判断。
两层信息不重复，最终仓位合理。
实际案例校验（验证四层管道的行为）：
━━━━━━━━━━━━━━━━━━━━━━━━

为什么用 ¼ Kelly 而非 ½ Kelly：
━━━━━━━━━━━━━━━━━━━━━━━━

概率100%主观估计，不可重复验证
5个情景无法穷举所有可能路径
情景之间存在相关性（宏观变量同时影响多个情景）
¼ Kelly意味着即使概率偏差一倍，仓位仍不致命

为什么用总回报而非年化IRR做Kelly：
━━━━━━━━━━━━━━━━━━━━━━━━

年化IRR压缩了回报的离散度
例：+179%和-14%的5年总回报，年化后变成+22.8%和-3.0%
用年化方差做分母会严重低估真实风险，导致Kelly虚高
Kelly公式的Var[r]需要反映你实际承受的回报波动

为什么不用二元Kelly：
━━━━━━━━━━━━━━━━━━━━━━━━

二元Kelly假设"赢b或亏100%"
股票最差情景不是亏光（CELH最差亏57%，HONA最差亏14%）
每个单独情景概率<50%时，二元Kelly对每个都输出0
多情景均值-方差Kelly: f*=E[r]/Var[r] 是正确的近似

指数压缩函数 Position = Cap × (1 − e^(−Adj/S)) 的设计：
━━━━━━━━━━━━━━━━━━━━━━━━
两个参数：

Cap = 10%：渐近线，单只个股的绝对上限。基于组合持有8-12只标的，10%是合理的集中度
S = 12%：斜率参数，控制曲线从0到Cap的攀升速度

S=12%的校准依据：

Adj Raw=10%（典型"标准conviction"标的）→ 仓位≈5.7%，接近半仓
Adj Raw=5%（低conviction/高方差/中等IRR标的）→ 仓位≈3.4%，合理的试探仓
Adj Raw=15%（强conviction + 高IRR标的）→ 仓位≈7.1%，明确的重仓信号
Adj Raw=20%+（极高conviction + 极高IRR标的）→ 仓位≈8.1%+，趋近但不到满仓

S参数的调整指南：

如果组合偏集中（6-8只），提高S到15%→曲线更缓→仓位更分散
如果组合偏分散（15+只），降低S到8-10%→曲线更陡→conviction差异更敏感
如果整体风险偏好更保守，降低Cap到8%

反向推算表的价值：
━━━━━━━━━━━━━━━━━━━━━━━━

把"该不该买"转化为"在什么价格买多少"——更有操作性
直接对接sell put策略：不同strike = 不同入场价 = 不同仓位
提前设定好分批建仓的价格梯度，消除情绪化决策
暴露"当前价格距离舒适区有多远"——比单一仓位数字更有信息量
新框架下，每个价格点的仓位都是连续函数输出，不存在人为跳变
Q因子在不同入场价下会变化（入场价越低→IRR越高→Q越大），形成双重加速效应

10% Cap的理由：
━━━━━━━━━━━━━━━━━━━━━━━━

组合有8-12个持仓（Barrick、sell puts、watchlist上的ZTS/IDXX/TDG等）
Kelly假设"唯一赌注"，多持仓时需要缩放
10%是集中组合中单只的合理上限
Kelly的真正价值是排序信号：Adj Raw大的标的仓位应该 > Adj Raw小的
Cap只防止极端情况，大多数标的的映射输出本来就在10%以下

置信度校准指南：
━━━━━━━━━━━━━━━━━━━━━━━━

0.80+ — 你跟踪了多年，有实证track record
0.70 — 深度研究过，理解商业模式和竞争格局
0.55-0.65 — 第一次看，或thesis依赖宏观/不可控变量
<0.50 — 基本是猜测，考虑是否值得投入研究时间

信息流总结（旧 vs 新）：
━━━━━━━━━━━━━━━━━━━━━━━━
旧（三层 + 阶梯）：
E[r],Var[r] → Kelly → ¼Kelly → ×Conf → Raw → min(Raw, f(E[IRR]))
问题：最后一步的f(E[IRR])是离散阶梯，且E[IRR]≈E[r]的单调变换，重复使用同一信息但丢掉方差
旧（三层 + 指数压缩，无Q）：
E[r],Var[r] → Kelly → ¼Kelly → ×Conf → Raw → Cap×(1−e^(−Raw/S))
问题：完全不考虑IRR绝对水平，低IRR+低方差标的获得过高仓位
新（四层）：
E[r],Var[r] → Kelly → ¼Kelly → ×Conf → Raw → ×Q(E[IRR]) → Adj Raw → Cap×(1−e^(−Adj/S))

Kelly和Q的输入相关但回答不同问题（效率 vs 门槛）
Q是连续函数，不产生离散跳变
Adj Raw同时编码了效率、置信度、绝对回报三维信息
压缩函数只做范围映射，不引入新信息源
